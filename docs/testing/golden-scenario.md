# Золотой сценарий (end-to-end сходимость)

Цель: один проверяемый сценарий «как часы»: закупка → склад → рейс → приёмка с недостачей → продажи (нал / долг) → **опционально** возврат и частичная оплата долга — и **сходимость остатков и денег**.

## Статус реализации

- **Реализовано (частично):** сквозной HTTP-тест сходимости граммов, выручки, **`financials`**, раздельно **нал/долг** в `sales` (`paymentKind: "debt"` в примере) — `apps/api/src/http/golden-scenario.flow.test.ts` (закупка → рейс → отгрузка → **недостача** → продажа → отчёт).
- **Ещё не в тесте:** смешанная оплата в золотом сценарии (есть в API `paymentKind: "mixed"`); возвраты; частичные оплаты долга.
- Тест гоняется в CI вместе с `pnpm test` (in-memory репозитории).

## Шаги сценария (черновик)

1. Закупка **5000 кг** (в граммах: `5_000_000`) по цене закупки X.
2. Поступление на склад: факт **4900 кг**, **100 кг** брак/списание на входе (уточнить документ).
3. Распределение по направлениям (если в объёме).
4. Рейс создан; отгрузка в рейс **3000 кг** (пример из текста заказчика).
5. Приёмка: принято **2900 кг** при отгруженных **3000** → недостача **100 кг**.
6. Продажи на стороне продавца: **нал / долг / смешанная** — суммы в копейках сходятся.
7. **Частичная оплата долга** — только после фиксации правил в `roles-and-permissions.md`.
8. **Возврат товара** — только после введения документа возврата в домене.

## Критерий успеха

- Отчёт по рейсу: отгружено / принято / недостача / продано / остаток / выручка / прибыль — **без расхождений** с движениями.
- Нет «плавающих» копеек и граммов из-за float.

## Связь с кодом

- Обязательный для регрессии in-memory тест: `apps/api/src/http/golden-scenario.flow.test.ts` (см. `pnpm test`).
- Отдельный PG/integration-файл — по необходимости; статус и пробелы к золотому сценарию ведутся в **`docs/implementation-status.md`**.
