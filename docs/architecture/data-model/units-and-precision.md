# Единицы измерения и точность

## Масса

- **Канон в БД и в доменной модели (целевое состояние):** масса хранится как **целое число граммов** (`grams`), без float.
- **Отображение:** кг с нужным числом знаков — только **представление** (UI, отчёты), не источник истины.
- **Почему:** исключает накопление ошибок float и расхождения с Excel из-за округления.

## Деньги

- Хранить **минорные единицы** (копейки) целым числом **или** `decimal(…)` в БД; в TypeScript — value object `Money` (уже в коде) с валютой.

## Миграция от текущего кода

В репозитории ранний класс `Batch` может быть в **килограммах (number)** — это **технический долг** до рефакторинга на `grams`. Новый код и новые таблицы — **сразу в граммах**, если нет явного указания обратного от заказчика.

## Согласование с глоссарием

В `business-glossary.md` учёт ведётся в кг и ящиках: **кг** в интерфейсе = `grams / 1000`; **ящики** — отдельное поле/движение, не смешивать с массой в граммах без явного коэффициента.
