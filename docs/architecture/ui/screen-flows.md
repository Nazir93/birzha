# Экранные сценарии для web и mobile

## Цель

Описать минимально необходимый набор экранов и пользовательских потоков, чтобы архитектура данных и документов сразу была связана с реальной работой сотрудников.

## Принцип интерфейсов

- `Web Admin` для офисных и управленческих операций
- `Mobile/PWA` для полевых сотрудников
- одна и та же бизнес-логика должна вызываться из обоих интерфейсов

## Web Admin

### 1. Дашборд руководителя
Показывает:
- остатки по складам
- активные рейсы
- продажи за день
- открытые долги
- проблемные синхронизации
- предупреждения по расхождениям

### 2. Справочники
Экраны:
- товары
- варианты товара
- калибры
- классы качества
- направления
- склады
- рынки
- клиенты
- сотрудники
- машины и маршруты

### 3. Закупка
Сценарий:
1. открыть список закупок
2. создать закупочную накладную
3. добавить строки товара
4. прикрепить фото накладной
5. отправить на подтверждение

### 4. Поступление на склад
Сценарий:
1. открыть неподтвержденные закупки
2. выбрать закупку
3. зафиксировать фактическое количество кг и ящиков
4. провести поступление
5. убедиться, что создана партия

### 5. Распределение по качеству и направлению
Сценарий:
1. открыть доступные партии на складе
2. выбрать партию
3. разделить по качеству
4. назначить направление
5. провести документ

### 6. Рейсы и логистика
Сценарий:
1. создать рейс
2. назначить фуру, водителя и маршрут
3. выбрать партии на погрузку
4. оформить отгрузку
5. отслеживать статус приемки и продаж

### 7. Продажи и долги
Офисные экраны:
- журнал продаж
- карточка клиента
- журнал долгов
- журнал оплат
- возвраты

### 8. Отчеты
Обязательные web-экраны:
- остатки по складам
- остатки по продавцам
- продажи по рейсам
- продажи по клиентам
- долги и просрочка
- итог по фуре

## Mobile/PWA для закупщика

### Экран 1. Мои закупки
- список последних закупок
- фильтр по дате и складу
- статус синхронизации

### Экран 2. Создать закупку
Поля:
- поставщик
- склад
- дата
- строки товара
- цена
- кг
- ящики
- фото накладной

### Экран 3. Детали закупки
- просмотр статуса
- просмотр принятых количеств
- история комментариев

## Mobile/PWA для кладовщика

### Экран 1. Поступления
- новые закупки к приемке
- черновики поступлений
- проведенные поступления

### Экран 2. Приемка на склад
Шаги:
1. выбрать закупку
2. проверить строки
3. скорректировать фактические количества
4. присвоить качество
5. провести поступление

### Экран 3. Погрузка
- активные задания на погрузку
- список партий
- подтверждение фактической загрузки

## Mobile/PWA для приемщика

### Экран 1. Мои рейсы
- список ожидаемых рейсов
- статус синхронизации

### Экран 2. Приемка рейса
Шаги:
1. открыть рейс
2. увидеть отгруженные позиции
3. ввести принятое количество
4. зафиксировать расхождения
5. приложить фото
6. подтвердить приемку

## Mobile/PWA для продавца

### Экран 1. Остаток на машине
Показывает:
- товар
- партия
- кг
- ящики
- цена продажи по умолчанию, если используется
- статус синхронизации

### Экран 2. Быстрая продажа
Шаги:
1. выбрать товар из доступного остатка
2. указать кг или ящики
3. указать цену
4. выбрать тип продажи
5. выбрать тип оплаты
6. сохранить продажу

### Экран 3. Продажа в долг
Шаги:
1. выбрать или создать клиента
2. указать товар и количество
3. указать сумму оплаты сейчас
4. система рассчитает долг
5. сохранить документ

### Экран 4. Оплата долга
Шаги:
1. открыть список долгов
2. выбрать клиента
3. указать сумму оплаты
4. сохранить платеж

### Экран 5. Списания
- списать порчу или нереализуемый товар
- указать причину
- приложить комментарий или фото

## Сквозные UI-правила

- в каждом журнале нужен статус синхронизации
- все документы должны показывать, проведены ли они на сервере
- при конфликте пользователь должен видеть причину и рекомендуемое действие
- остатки должны обновляться после каждого подтвержденного действия

## Обязательные уведомления

- товар принят по рейсу
- продажа синхронизирована
- конфликт синхронизации
- превышение лимита долга
- закрытие рейса
