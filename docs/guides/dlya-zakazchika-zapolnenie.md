# Как заполнять систему «Биржа»: инструкция для заказчика и сотрудников

Документ для **тех, кто вводит данные в веб-интерфейсе**. Цель: один раз понять **порядок действий** и **смысл полей**, чтобы цифры в отчёте сходились с реальностью.

---

## 1. О чём система одной фразой

**Товар закупают → кладут на склад → везут рейсом → продают (или фиксируют недостачу).**  
Все массы — в **килограммах**, деньги в отчёте показываются в **рублях** (внутри считаются в копейках).

---

## 2. Общий порядок работы (не переставляйте шаги)

| № | Что делаем | Где в интерфейсе |
|---|------------|------------------|
| 1 | Войти под своей учётной записью | Страница **«Вход»** (`/login`) |
| 2 | При необходимости убедиться, что сервер «включён» | Вкладка **«Служебное»** — в JSON должно быть много `"enabled"` (для администратора/менеджера) |
| 3 | Создать **рейс** (машина / маршрут) | **«Отчёты и рейсы»** — блок «Создать рейс» |
| 4 | Зафиксировать **закупку** | Вариант А: **«Накладная»** (как бумажная накладная). Вариант Б: **«Операции»** — «Создать партию» без накладной |
| 5 | Движения по рейсу: отгрузка → при необходимости недостача → продажа | **«Операции»** — блоки 3–5 |
| 6 | Проверить итоги | **«Отчёты и рейсы»** — выбрать рейс, смотреть таблицы и суммы |

**Правило:** сначала должен существовать **рейс**, если вы грузите товар **в рейс**. Накладная или партия создаёт **остаток на складе**; отгрузка **переносит килограммы** в рейс.

---

## 3. Вкладка «Отчёты и рейсы»

### 3.1. Создать рейс

| Поле | Как заполнять |
|------|----------------|
| **Номер рейса** * | Как в вашем учёте, например `Ф-2026-001`. Обязательно. |
| **ID рейса** | Можно **оставить пустым** — система сама присвоит технический номер (UUID). Заполняют редко, если нужна жёсткая связь с другой системой. |

**Кнопка «Создать рейс»** — после успеха рейс появится в списке и в выборе на «Операциях».

**Удалить рейс:** на странице «Отчёты и рейсы», после выбора рейса, если по нему **ещё нет** отгрузок, продаж и недостач (нули в отчёте), доступна кнопка **«Удалить пустой рейс»** (нужны права логиста, менеджера или администратора). Если по рейсу уже были операции — удаление через интерфейс **недоступно** (учётная целостность).

### 3.2. Отчёт по рейсу

| Действие | Смысл |
|----------|--------|
| Выбрать **рейс** в списке | Подтягивается отчёт: отгрузки, продажи, недостачи, деньги, строки по партиям. |
| **Печать** | Печать текущего отчёта. |
| **Скачать CSV** (если есть) | Таблица по партиям для Excel. |

**Как понять, что всё сошлось:** итоговые килограммы вверху должны совпадать с суммой по партиям внизу; если система показывает предупреждения о расхождении — проверьте ввод в «Операциях».

---

## 4. Вкладка «Накладная» (закупочная накладная)

Используйте, когда закупка оформляется **как на бумаге**: шапка документа + строки по **калибру** (виду товара).

### 4.1. Шапка

| Поле | Как заполнять |
|------|----------------|
| **Номер документа** * | Номер с бумажной накладной, например `НФ-100`. |
| **Дата** * | Дата документа, формат **ГГГГ-ММ-ДД** (календарь в поле). |
| **Склад** * | Выберите из списка — это склад **поступления**. |
| **ID документа** | Обычно **пусто** (сервер сам задаст id). |
| **Поставщик** | По желанию, текстом. |
| **Покупатель / подпись** | По желанию. |
| **Доп. расходы, коп.** | Целое число копеек на документ сверх строк (если не используете — `0`). |

### 4.2. Строки (каждая строка = одна партия на складе)

| Поле | Как заполнять |
|------|----------------|
| **Калибр** * | Выбор из справочника (виды/коды товара). Не придумывайте название вручную — только из списка. |
| **Кг** * | Масса по строке, можно с десятичной запятой или точкой. |
| **Короба** | При необходимости, целое число. |
| **₽/кг** * | Закупочная цена за килограмм **в рублях** (как в колонке «Цена» на накладной). |
| **Сумма, коп.** * | Сумма строки **в копейках**. Должна совпадать с **кг × цена за кг** (допуск на сервере ±1 копейка). Кнопка **«=кг×цена»** подставляет расчёт. |

**Кнопки:** **«Добавить строку»** — ещё одна позиция; **«Создать накладную»** — отправить документ.

**После успеха:** по каждой строке создаётся **партия** на выбранном складе; дальше с ней работают в **«Операциях»** (отгрузка в рейс и т.д.).

---

## 5. Вкладка «Операции»

Блоки пронумерованы — в типичном сценарии идут **сверху вниз**.

### 5.1. Создать партию (если **не** использовали накладную)

| Поле | Как заполнять |
|------|----------------|
| **ID партии** | Можно пусто — подставится при отправке. |
| **ID закупки** | Можно пусто — подставится. |
| **totalKg** * | Вся закупленная масса по этой партии, кг. |
| **pricePerKg** * | Закупочная цена **руб/кг** (для себестоимости в отчёте). |
| **distribution** | **on_hand** — сразу на складе; **awaiting_receipt** — ждёт оприходования (тогда нужен блок 2). |

### 5.2. Оприходовать на склад

Только если партия была **awaiting_receipt**.

| Поле | Смысл |
|------|--------|
| **batchId** | Скопируйте из таблицы партий выше на странице. |
| **kg** | Сколько килограмм **приняли фактически** на склад. |

### 5.3. Отгрузить в рейс

| Поле | Смысл |
|------|--------|
| **batchId** | Партия, с которой везёте. |
| **tripId** | Выберите **рейс** из списка (сначала создайте рейс в «Отчётах»). |
| **kg** | Сколько килограмм **отгрузили в этот рейс**. |

### 5.4. Недостача по рейсу (при приёмке)

| Поле | Смысл |
|------|--------|
| **batchId**, **tripId** | Та же пара. |
| **kg** | Недостача **в килограммах** (фиксируется по правилам вашего учёта). |
| **reason** | Кратко причина (текст обязателен для сервера). |

### 5.5. Продажа с рейса

| Поле | Смысл |
|------|--------|
| **batchId**, **tripId** | Партия и рейс. |
| **kg** | Сколько продаёте **из того, что в пути по этому рейсу**. |
| **saleId** | Можно пусто — система создаст. |
| **pricePerKg** | Цена продажи **руб/кг** (влияет на выручку в отчёте). |
| **Оплата** | Наличные / долг / смешанная — по факту сделки. При «смешанной» укажите сумму наличными в копейках, как требует форма. |
| **Клиент** | Либо выберите **контрагента** из справочника, либо введите **подпись** (как в договоре/накладной) — для столбца «по клиентам» в отчёте. |

При необходимости контрагента можно **создать** в том же разделе (кнопка добавления — если она есть в вашей версии экрана).

## 6. Вкладка «Офлайн-очередь»

Для работы **без стабильного интернета**: действия складываются в очередь и отправляются кнопкой **«Синхронизировать»**.  
Для первичного обучения заказчика достаточно знать, что **основной сценарий** — через обычные вкладки выше.

---

## 7. Типичные ошибки и как их избегать

| Проблема | Что делать |
|----------|------------|
| «Не сходится сумма строки» на накладной | Пересчитать **копейки** = кг × руб/кг × 100, либо нажать **«=кг×цена»**. |
| Нет рейса в списке отгрузки | Сначала **создайте рейс** в «Отчётах». |
| Отчёт пустой | Не выбран рейс **или** по рейсу ещё не было отгрузок/продаж. |
| Очень большой минус в «валовой прибыли» | Проверьте **цену продажи за кг** и **закупочную цену партии** — тестовый ввод `1 ₽/кг` при закупке `40 ₽/кг` даст отрицательную прибыль **по смыслу**, не как «сломалось». |
| Не видно вкладок | У разных ролей разный набор вкладок (например, бухгалтер не видит «Операции»). Вход под **admin** даёт полный набор для проверки. |

---

## 8. Краткая шпаргалка «что за чем»

```
Вход → (по желанию Служебное / meta)
     → Создать РЕЙС
     → НАКЛАДНАЯ или ПАРТИЯ на складе
     → Отгрузка в РЕЙС
     → Недостача (если было)
     → Продажа с рейса
     → ОТЧЁТ по рейсу — проверка цифр
```

---

## 9. Для заказчика: что «подписать» с поставщиком системы

Чтобы внедрение не спорило с документами, заранее определите:

1. **Единый ли справочник калибров** (виды товара) ведёте вы или поставщик данных.  
2. **Обязательны ли склады** в 1С-смысле или достаточно одного склада на старте.  
3. **Кто отвечает** за ввод накладной и кто за отгрузку в рейс (часто разные люди).
